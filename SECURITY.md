# Security Policy

## Supported Versions

At the moment, SafeSkill is maintained on the latest mainline version.

| Version | Supported |
|---------|-----------|
| 0.4.x   | ✅ |
| < 0.4   | ❌ |

## Reporting a Vulnerability

If you discover a serious vulnerability in SafeSkill itself, please avoid posting full exploit details publicly right away.

Instead, open a private report to the maintainer through GitHub security reporting if available, or contact the maintainer directly through the GitHub profile:
- https://github.com/AIPMAndy

Please include:
- affected version
- reproduction steps
- expected impact
- any proof of concept if safe to share

## Scope

This policy applies to:
- the SafeSkill scanner code
- CI/CD and release workflow security relevant to the repo
- packaging and installation paths

It does **not** mean every rule-detection miss is automatically a vulnerability in SafeSkill. False negatives and false positives can also be filed as normal issues if they are not security-sensitive.
